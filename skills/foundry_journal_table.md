# Skill: Генерация HTML-таблиц для журнала FoundryVTT

## Когда использовать
- Пользователь просит создать таблицу / каталог / список для FoundryVTT
- Упоминается "журнал", "journal page", "Foundry таблица"
- Нужно конвертировать Obsidian-таблицу в HTML для Foundry
- Запрос "сгенерируй HTML для журнала", "сделай таблицу для Foundry"
- Нужно обновить существующую HTML-таблицу журнала новыми данными

---

## Контекст (что читать)

**Обязательно:**
- Obsidian-файл с данными (markdown-таблица)
- Этот скилл — для стилевых констант и шаблонов

**Опционально:**
- Существующие HTML-таблицы журнала (для сверки стиля, если есть сомнения)

---

## Стилевые константы

Единая цветовая палитра для всех таблиц журнала:

```
HEADER_BG         = #3A2800        — фон заголовка таблицы (thead)
HEADER_TEXT        = #E8C870        — текст заголовков (th)
NAME_TEXT          = #D4C4A0        — названия / имена (первая колонка)
ACCENT_TEXT        = #C8A050        — цены, числа, метки
BODY_TEXT          = #9A8878        — основной текст описания
EMPHASIS_TEXT      = #B8A080        — выделенный текст (em/курсив внутри описания)
BORDER_THIN        = #5A4020        — тонкий бордер (под заголовком, под заголовком категории)
BORDER_ROW         = #3A2800        — бордер между строками
ACCENT_LINE        = #8B6914        — акцентная линия (заголовок страницы, инфоблок)
INFO_BG            = rgba(139,105,20,0.15) — фон инфоблока
INFO_TEXT          = #B8956A        — текст инфоблока
ALT_ROW_BG         = rgba(255,255,255,0.02) — фон чётных строк (чередование)
```

## Шаблоны HTML-элементов

### 1. Заголовок страницы

```html
<p><span style="text-align: center; font-family: Georgia, serif; border-bottom: 2px solid #8B6914; padding-bottom: 8px; color: #E8C870">{ЭМОДЗИ} {ЗАГОЛОВОК}</span></p>
```

**Пример:** `☕ Каталог безалкогольных напитков — Ванфьёр`

---

### 2. Информационный блок

Используй для пояснений, легенды, системы классов и т.п.

```html
<p style="background: rgba(139,105,20,0.15); border-left: 4px solid #8B6914; padding: 8px 12px; font-size: 0.85em; color: #B8956A"><strong style="color:#E8C870">{ЗАГОЛОВОК БЛОКА}:</strong>   {СОДЕРЖИМОЕ}</p>
```

**Пример:** `Классы заведений: I — простое | II — среднее | III — дорогое`

---

### 3. Подзаголовок категории

```html
<p><span style="font-family: Georgia, serif; color: #E8C870; border-bottom: 1px solid #5A4020; margin-top: 24px">{ЭМОДЗИ} {НАЗВАНИЕ КАТЕГОРИИ}</span></p>
```

**Пример:** `🍇 Виноградное`

---

### 4. Таблица (thead)

```html
<table style="width:100%;border-collapse:collapse;font-size:0.9em">
    <thead>
        <tr style="background:#3A2800">
            <th style="padding:6px 10px;text-align:left;width:{W1}%;color:#E8C870;border-bottom:1px solid #5A4020">{КОЛОНКА 1}</th>
            <th style="padding:6px 10px;text-align:left;width:{W2}%;color:#E8C870;border-bottom:1px solid #5A4020">{КОЛОНКА 2}</th>
            <th style="padding:6px 10px;text-align:center;width:{W3}%;color:#E8C870;border-bottom:1px solid #5A4020">{КОЛОНКА 3}</th>
            <th style="padding:6px 10px;text-align:left;color:#E8C870;border-bottom:1px solid #5A4020">{КОЛОНКА 4}</th>
        </tr>
    </thead>
    <tbody>
        {СТРОКИ}
    </tbody>
</table>
```

**Ширины по умолчанию:**
- 4 колонки: 28% / 14% / 8% / auto
- 3 колонки: 30% / 15% / auto
- 2 колонки: 30% / auto

---

### 5. Строка таблицы (нечётная — без фона)

```html
<tr>
    <td style="padding:6px 10px;font-weight:bold;color:#D4C4A0;border-bottom:1px solid #3A2800"><strong>{НАЗВАНИЕ}</strong></td>
    <td style="padding:6px 10px;color:#C8A050;border-bottom:1px solid #3A2800">{ЗНАЧЕНИЕ}</td>
    <td style="padding:6px 10px;text-align:center;color:#C8A050;border-bottom:1px solid #3A2800">{МЕТКА}</td>
    <td style="padding:6px 10px;border-bottom:1px solid #3A2800;color:#9A8878"><em style="color:#B8A080">{АКЦЕНТ}</em> {ОПИСАНИЕ}</td>
</tr>
```

### 6. Строка таблицы (чётная — с фоном)

```html
<tr style="background:rgba(255,255,255,0.02)">
    <!-- содержимое идентично нечётной строке -->
</tr>
```

### 7. Строка с @UUID (если предмет существует в Foundry)

```html
<td style="padding:6px 10px;font-weight:bold;color:#D4C4A0;border-bottom:1px solid #3A2800"><strong>@UUID[Item.{ID}]{ {НАЗВАНИЕ} }</strong></td>
```

## Workflow

### Шаг 1: Определи тип контента

Спроси или определи из контекста:
- Какой пресет колонок? (товарный каталог, NPC, квест, инвентарь, пользовательский)
- Заголовок страницы и эмодзи
- Нужен ли информационный блок (легенда, пояснения)?
- Есть ли подкатегории (секции с подзаголовками)?

### Шаг 2: Прочитай данные

Прочитай указанный Obsidian-файл (.md) через MCP или Read tool.
Найди markdown-таблицы и подзаголовки (##/###).

### Шаг 3: Парси markdown → структура данных

Для каждой таблицы:
1. Извлеки заголовки колонок
2. Извлеки строки данных
3. Определи подкатегорию (из ближайшего заголовка ##/### над таблицей)

### Шаг 4: Применяй правила конвертации

**Текст:**
- `**Жирный**` → `<strong>Текст</strong>`
- `[[Ссылка]]` → просто текст (убрать вики-скобки)
- `[[Ссылка|Текст]]` → просто Текст

**Классы заведений (если есть колонка Класс):**
- `1` → `I`
- `1–2` → `I–II`
- `2` → `II`
- `2–3` → `II–III`
- `3` → `III`

**Объединение колонок:**
- Если есть отдельные колонки «Вкус» и «Описание» — объединить в одну:
  `<em style="color:#B8A080">{Вкус}.</em> {Описание}`
- Вкус — сокращай до ключевой фразы (1-2 предложения), описание — тоже кратко

**@UUID:**
- Если пользователь предоставляет UUID — используй формат `@UUID[Item.{ID}]{Название}`
- Если UUID нет — просто `<strong>Название</strong>`

**Чередование строк:**
- Нечётные строки (1, 3, 5...): `<tr>`
- Чётные строки (2, 4, 6...): `<tr style="background:rgba(255,255,255,0.02)">`
- Счётчик сбрасывается в каждой новой подкатегории

### Шаг 5: Собери HTML

Порядок сборки:
1. Заголовок страницы
2. Информационный блок (если нужен)
3. Для каждой подкатегории:
   - Подзаголовок категории
   - Таблица (thead + tbody со строками)
4. Выведи весь HTML в чат в блоке кода (```) для удобного копирования
