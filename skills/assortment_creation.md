# Skill: Создание ассортимента товаров и услуг

## Когда использовать
- Пользователь просит создать ассортимент / каталог / меню для региона
- Нужно заполнить таверну, магазин, аптеку, кузницу товарами
- Запрос «придумай еду / напитки / товары для [региона]»
- Расширение существующего ассортимента новыми категориями
- Создание ассортимента для нового региона по аналогии с существующим

---

## Контекст (что читать)

**Обязательно:**
- Файлы целевого региона (`0.Лор/Мир/Атлас/[регион]`) — климат, ресурсы, культура
- Существующий ассортимент этого региона (если есть) — для согласованности цен и стиля

**Опционально:**
- Файлы соседних регионов — для привозных товаров
- `1.Черновик/` — черновые наработки по региону
- Другие ассортименты — для сверки формата и ценообразования

**Эталонные файлы (формат):**
- `0.Лор/Предметы/Товары/Алкоголь/Ассортимент таверн Ванфьёра.md`
- `0.Лор/Предметы/Товары/Безалкогольные напитки/Ассортимент б.а. напитков Ванфьёр.md`

## Шаблон файла

Каждый файл ассортимента строится по единой структуре:

### 1. YAML Frontmatter

```yaml
---
type: REFERENCE
subtype: tavern_assortment    # или shop_assortment, smithy_assortment
region: {Регион}
kingdom: {Королевство}
status: wip
tags:
  - ассортимент
  - {тип заведения}            # таверна, магазин, аптека...
  - {тип товаров}              # напитки, еда, оружие...
  - {регион lowercase}
created: '{YYYY-MM-DD}'
author: claude
---
```

### 2. Changelog

```markdown
> [!changelog]-
> **{YYYY-MM-DD} (claude):** Создан файл. {Описание содержимого}.
```

Правила:
- Новые записи добавляются **сверху**
- Формат даты: `YYYY-MM-DD`
- Автор в скобках: `(claude)` или имя пользователя
- Краткое описание изменений: что добавлено, сколько позиций

### 3. Заголовок и описание

```markdown
# {Название ассортимента}

Справочник {тип товаров} региона [[{Регион}]] для случайных таблиц FoundryVTT.
```

**Примеры названий:**
- `Ассортимент таверн и постоялых дворов Ванфьёра`
- `Ассортимент безалкогольных напитков Ванфьёра`
- `Ассортимент кузнечной лавки Сейры`

### 4. Классы заведений

Стандартный блок, одинаковый для всех файлов:

```markdown
## Классы заведений

Каждая позиция имеет метку **Класс**, определяющую, в каком заведении она доступна:
- **1** — простое (портовая таверна, горная хижина пастуха)
- **2** — среднее (городской трактир)
- **3** — дорогое (элитное заведение, приём у аристократа)

Позиции с диапазоном (1–2, 2–3) встречаются в заведениях обоих классов.
```

> **Адаптация:** для магазинов замени примеры:
> - **1** — придорожный лоток, деревенская лавка
> - **2** — городской магазин
> - **3** — элитная лавка, дворцовый поставщик

### 5. Справочные таблицы (опционально)

Если ассортимент связан с уникальными производителями, фракциями или брендами — добавь справочную таблицу после классов.

**Пример (семьи виноделов Ванфьёра):**

```markdown
## Четыре семьи виноделов

| Семья | Характер | Репутация |
|-------|----------|-----------|
| **Бьёрнхольм** | Старейшая династия, консерваторы | Надёжность |
| **Сольвейг** | Экспериментаторы, молодая семья | Дерзость |
```

### 6. Оглавление (Категории)

```markdown
## Категории

- [[#Название категории 1]]
- [[#Название категории 2]]
- [[#Промпты для изображений]]
```

Между «Категории» и первой категорией — разделитель `---`.

### 7. Таблицы товаров

Каждая категория — заголовок H2. Подкатегории (если нужны) — H3.

## Формат таблицы

### Колонки

```markdown
| # | Название | Цена ({единица}) | Класс | Вкус | Описание |
|---|----------|------------------|-------|------|----------|
```

**Единица цены** зависит от типа товара:
- Напитки: `кружка`, `бокал`, `стопка`
- Еда: `порция`, `блюдо`, `штука`
- Товары: `штука`, `за фунт`, `за дюжину`

### Нумерация

**Сквозная** по всему файлу. Не сбрасывается в подкатегориях.

### Цены

Валюты D&D:
- **мм** — медные монеты (самые мелкие)
- **сс** — серебряные монеты (1 сс = 10 мм)
- **зм** — золотые монеты (1 зм = 10 сс = 100 мм)

Формат: `1 мм`, `5 мм`, `1 сс`, `3 сс`, `1 зм`, `2 зм 5 сс`

**Ориентиры ценообразования (таверна/еда):**

| Класс | Напиток | Блюдо | Ориентир |
|-------|---------|-------|----------|
| 1 | 1–5 мм | 3–8 мм | Дневной заработок чернорабочего ≈ 1 сс |
| 1–2 | 5 мм – 2 сс | 5 мм – 3 сс | Стандарт для большинства жителей |
| 2 | 2–5 сс | 2–6 сс | Горожанин среднего достатка |
| 2–3 | 5–10 сс | 5 сс – 1 зм | Заметная трата для среднего |
| 3 | 1+ зм | 1+ зм | Роскошь, статусное потребление |

### Классы заведений

Используй арабские цифры в таблице: `1`, `2`, `3`, `1–2`, `2–3`

> При конвертации в HTML для FoundryVTT (skill `foundry_journal_table`) они заменяются на римские: I, II, III, I–II, II–III.

### Стиль колонки «Вкус»

Сенсорное описание от первого впечатления. 1–3 предложения.

**Обязательно включи:**
- Вкусовые ноты (сладкий, горький, кислый, солёный, умами)
- Текстуру (густой, жидкий, бархатистый, маслянистый)
- Температуру (если важна)
- Послевкусие или финиш (опционально)

**Стиль:** образный, с метафорами. Не сухой перечень — читатель должен «почувствовать».

✅ `Горячий мёд, имбирь, корица. Жидкий огонь, но мягкий — согревает до кончиков пальцев`
❌ `Сладкий, пряный, горячий напиток`

### Стиль колонки «Описание»

Нарративное описание: что это, откуда, кто пьёт/ест, культурный контекст. 1–3 предложения.

**Включай:**
- Производство/происхождение
- Кто типичный потребитель
- Культурный контекст или поговорку
- Где подают / где не найти

Используй `[[Wikilinks]]` для ссылок на локации и NPC.

✅ `Кисломолочный напиток из овечьего молока. Пастухи берут во фляге на весь день — не портится и утоляет и жажду, и голод`
❌ `Кисломолочный напиток. Популярен у пастухов`

## Правила наполнения

### Принцип «от региона»

Ассортимент строится на основе региона, а не наоборот:

1. **Климат и география** → доступное сырьё
2. **Культура и этнос** → способы приготовления, традиции
3. **Торговые связи** → привозные товары (дороже местных)
4. **Уровень развития** → сложность кулинарии

**Пример (Ванфьёр):**
- Горный полуостров, тёплый климат → виноград, горные травы, рыба, баранина
- Нордическая культура → медовуха, сбитень, грубая простая кухня
- Порт Бауесх → привозной чай, кофе, ром, специи
- Виноделие → горельник (дистиллят), бергвинд (настойки)

### Баланс классов

В каждой категории стремись к распределению:
- **Класс 1 и 1–2:** 40–50% позиций (основа ассортимента)
- **Класс 2 и 2–3:** 30–40% позиций (средний сегмент)
- **Класс 3:** 10–20% позиций (редкости и роскошь)

### Баланс местного и привозного

- **70–80%** — местные товары (из ресурсов региона)
- **20–30%** — привозные (дороже, с пометкой откуда)

### Количество позиций

Рекомендуемое количество на категорию:
- Маленькая категория: 5–10 позиций
- Средняя: 10–18 позиций
- Большая: 18–25 позиций

Ориентир: достаточно для случайной таблицы d10/d12/d20 в FoundryVTT.

## Промпты для изображений

Секция промптов идёт в конце файла, после всех таблиц.

### Структура секции

```markdown
## Промпты для изображений

Индивидуальный промпт для каждого предмета. Без названий на этикетках. Технические параметры отдельной строкой.

---

#### 1. {Название}

\```
{Описание на русском — 2–4 строки: предмет, посуда/подача, детали, атмосфера}

--ar 1:1, dnd art, digital art, gray background
\```

#### 2. {Название}
...
```

### Правила промптов

1. **Язык:** русский (кроме технической строки)
2. **Без названий** товаров/брендов/семей в тексте промпта
3. **Серый фон** — всегда `gray background`
4. **Соотношение сторон:** `--ar 1:1` для всех товаров
5. **Технические параметры** — отдельная последняя строка: `--ar 1:1, dnd art, digital art, gray background`
6. **Код-блок** — каждый промпт обёрнут в ``` для одного клика копирования
7. **Нумерация** совпадает с таблицей (сквозная)
8. **Без посторонних элементов** — только товар и относящиеся к нему предметы (посуда, гарнир, сервировка)

### Что описывать в промпте

| Тип товара | Описывай | Не описывай |
|-----------|----------|-------------|
| Напиток | Посуда, цвет жидкости, текстура, пар, гарнир | Людей, интерьер, руки |
| Блюдо | Тарелка/миска, еда, гарнир, пар, текстура | Столы, людей, интерьер |
| Товар/предмет | Предмет, материал, упаковка, детали | Руки, людей, магазин |

## Workflow

### Шаг 1: Сбор контекста

1. Уточни **регион** и **тип заведения** (таверна, магазин, кузница)
2. Прочитай файлы региона — климат, ресурсы, культура, торговые связи
3. Проверь существующий ассортимент региона — для согласованности

### Шаг 2: Планирование категорий

1. Предложи список категорий (H2 секции)
2. Оцени количество позиций на категорию
3. Определи подкатегории (H3), если нужны
4. Определи единицу цены для заголовка таблицы
5. Согласуй с пользователем

### Шаг 3: Наполнение таблиц

Для каждой категории:
1. Придумай позиции, следуя принципу «от региона»
2. Заполни все 6 колонок: #, Название, Цена, Класс, Вкус, Описание
3. Проверь баланс классов и соотношение местного/привозного
4. Сквозная нумерация через весь файл
5. Выведи таблицу пользователю на проверку перед записью

### Шаг 4: Генерация промптов

После утверждения таблиц:
1. Создай промпт для каждой позиции
2. Формат: русский текст + `--ar 1:1, dnd art, digital art, gray background`
3. Без названий товаров, серый фон, код-блоки
4. Нумерация совпадает с таблицей

### Шаг 5: Запись в файл

1. Собери файл по шаблону (frontmatter → changelog → заголовок → классы → опц. справочные таблицы → категории TOC → таблицы → промпты)
2. Путь: `0.Лор/Предметы/Товары/{Подпапка}/{Название файла}.md`
3. Запиши через MCP Obsidian

### Шаг 6: Changelog

Добавь запись в changelog:
```
> **{YYYY-MM-DD} (claude):** Создан файл. {Краткое описание: что, сколько позиций}.
```

Структура папок:
```
0.Лор/Предметы/Товары/
├── Алкоголь/
│   └── Ассортимент таверн Ванфьёра.md
├── Безалкогольные напитки/
│   └── Ассортимент б.а. напитков Ванфьёр.md
├── Горячие блюда/
│   └── ...
├── Закуски/
│   └── ...
└── {Новая категория}/
    └── {Ассортимент ... Ванфьёр.md}
```

**Правило:** одна категория = одна папка = один файл на регион.

## Чеклист

### Структура файла
- [ ] YAML frontmatter заполнен (type, subtype, region, kingdom, status, tags, created, author)
- [ ] Changelog создан с датой и описанием
- [ ] Заголовок H1 + описание со ссылкой на регион
- [ ] Блок «Классы заведений» на месте
- [ ] TOC «Категории» со ссылками на все секции + промпты
- [ ] Разделители `---` между секциями

### Таблицы
- [ ] Все колонки: #, Название, Цена, Класс, Вкус, Описание
- [ ] Сквозная нумерация (не сбрасывается в подкатегориях)
- [ ] Цены в формате D&D (мм / сс / зм)
- [ ] Классы: 1, 1–2, 2, 2–3, 3 (с em-dash)
- [ ] Баланс классов: ~40% низкие, ~40% средние, ~20% высокие
- [ ] Вкус: сенсорный, образный, 1–3 предложения
- [ ] Описание: нарративное, с контекстом, 1–3 предложения
- [ ] Wikilinks для локаций

### Промпты
- [ ] Промпт для каждой позиции (количество = количеству строк в таблицах)
- [ ] Нумерация совпадает с таблицей
- [ ] Русский текст, без названий товаров/брендов
- [ ] Тех. параметры отдельной строкой: `--ar 1:1, dnd art, digital art, gray background`
- [ ] Обёрнуты в код-блоки
- [ ] Серый фон, только товар и сопутствующие предметы

### Контент
- [ ] Товары логичны для региона (климат, ресурсы, культура)
- [ ] Есть местные И привозные позиции
- [ ] Нет анахронизмов (фэнтези-средневековье)
- [ ] Нет дублирования с существующими файлами ассортимента региона
